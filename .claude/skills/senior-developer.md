## 시니어 개발자 Skill 규약 (실무 리딩 및 품질 관리)
시니어 개발자는 코드 품질과 실행 단의 최종 책임자입니다. 에이전트는 시니어와 대화할 때 아래 기술적 상세 지침을 따릅니다.

코드 리뷰 자동화 (Reviewer): 모든 코드 제안 시 "7. 코드 규칙(ConfigureAwait 금지)" 및 "DBA 규칙(FK 금지)" 위반 여부를 시니어 개발자의 시각에서 먼저 검토하여 보고한다.

추상화 및 재사용성: 중복 코드를 방지하기 위해 인터페이스(I) 기반의 설계와 공통 모듈화를 최우선으로 제안한다.

기술적 병목 해결: 개발자 2명이 해결하지 못하는 복잡한 알고리즘이나 동시성 문제(Async/Lock)에 대해 시니어 수준의 심도 있는 솔루션을 제공한다.

멘토링 모드: 개발자들에게 코드를 줄 때 단순히 결과물만 주는 것이 아니라, 왜 이런 패턴(SOLID 등)을 사용했는지 근거를 설명하여 팀 역량 향상을 돕는다.

자원 관리 감독: IDisposable, using, StringBuilder 사용 등 실무적인 성능 저하 요소를 엄격하게 감시한다.

## [DBA & 데이터베이스 설계 준칙]
데이터베이스와 관련된 모든 설계 및 쿼리 제안 시 아래 DBA 규칙을 절대적으로 준수한다.

외래 키(FK) 사용 금지: 데이터베이스 물리 계층에서의 FK(Foreign Key) 생성을 금지한다.

데이터 무결성은 DB 제약 조건이 아닌 애플리케이션 로직(C# 서비스 레이어)에서 검증하도록 코드를 제안한다.

에이전트는 테이블 설계 제안 시 CONSTRAINT FK_... 구문을 포함하지 않는다.

인덱스 최적화: 물리적인 관계 제약이 없으므로, 조인 성능 유지를 위해 연관 컬럼에 대한 인덱스(Index) 설계를 더욱 철저히 제안한다.

데이터 정합성: 삭제나 수정 시 연관 데이터를 처리하는 로직(Soft Delete 등)을 시니어 개발자와 개발자에게 명확히 가이드한다.
